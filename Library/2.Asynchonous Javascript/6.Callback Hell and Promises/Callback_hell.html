Callback Hell

pour appeler plusieurs documents Json, qui sont dans le dossier, on passe un second paramètre, en plus du callback

l'argument est resource

le resource permet de récupérer les éléménts de tous les fichiers

on passe ensuite le resource comme argument dans la requete

on peut ensuite nester plusieurs fois la même méthode en une seule fois, afin de faire plusieurs requêtes

le problème est quand on fait trop de nesting, le code devient illisible

pour corriger le problème, il faut utiliser les Promises

Promises

lien mozilla Promise

une Promise, permet d'éviter de nester trop de fonctions

une promise va avoir deux issues, une positive (resolve) et une négative (reject)

il faut prévoir les deux cas, et prévoir ce que la fonction va donner dans les deux

pour créer la promise, il faut:

  • créer une fonction
  • faire un return new Promise dans la première fonction puis créer la fonction associée
  • passer les deux paramètres resolve et reject
  • prévoir ce que font les resolve et reject

quand on appelle la fonction, il faut appeler la fonction, et ajouter la méthode .then pour appeler la callback function dans le then, puis prévoir ce qui va se passer dans chaque cas

une autre façon d'écrire la promise, est avec la méthode .catch
