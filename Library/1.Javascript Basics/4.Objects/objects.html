Objects

un objet a des propriétés et peut avoir des méthodes

un objet fonctionne avec un système de key value

on peut insérer un array dans l'objet

on peut récupérer les propriétés de l'object en les appelant, ex:user.name

on peut également récupérer la propriété avec la syntaxe user['name']

on peut mettre à jour les valeurs en modifiant directement

pour sortir une clé d'un objet dans une méthode, on utilise this, suivi de l'objet que l'on veut sortir

le this fait référence à l'objet quand il est utilisé dans une méthode

possibilité de refacto la syntaxe d'une fonction régulière

on peut stocker des objets dans une array, comme un nom d'auteur et une année pour un livre

Math object

Math est par définition un objet créé par JS, qui a de nombreuses méthodes prédéfinies

math function, link mozilla

utiliser Math pour y accéder

utiliser la méthode .random pour créer un nombre aléatoire

Références

les objects, array, functions, dates sont tous des références, alors que les nombre, boolean, string, null, undefined et symbol sont des primitives

cela influe sur la façon de les stocker et de les utiliser

les primitives sont stockés dans le "stack", et les références dans le "heap". Heap à plus de place dispo pour stocker

cela influe aussi la façon dont ils sont modifiables. Si on crée un primitive, qu'on lui assigne une valeur, puis qu'on en crée une copie, les deux auront la même valeur

cependant si on modifie l'original, cela ne va pas modifier la copie, on se retrouvera avec par exemple:

user = "coco"

user1 = user2

les deux users on coco comme string, mais si je change user1 en "erlé", seul user1 change, car les deux élements ne pointent pas vers la même chose à l'endroit où ils sont stockés

ils sont considérés comme deux valeurs différentes

à l'inverse, si je crée un objet, avec un nom et un age, si je crée un original et une copie, si je modifie l'original, alors je modifie la copie
