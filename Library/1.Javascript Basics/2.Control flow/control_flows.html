Loops

For loops

pour créer une boucle, on utilise le for

il faut une varaible d'initialisation, puis la condition, et une expression finale qui s'exécute à la fin du code

la condition peut également être une variable avec une méthode

While loops

même principe que le for, mais syntaxe différente

l'exemple permet de printer le nom du pokémon en fonction de sa position dans l'array

do while loop

c'est une extension du while

le do while crée une boucle qui exécute une instruction jusqu'à ce qu'une condition ne soit plus vérifiée

Conditionnal

If, else if et multiple conditions

si la condition est remplie, éxécute le code

voir la syntaxe pour le if else, et mulitple condition

pour le else if, on doit insérer les conditions par ordre décroissant si on manipule des chiffres

Opérateurs logiques

le && renvoie le code si deux conditions sont true, ou équivalent à true

on peut multiplier les conditions, le && ne vaudra que pour la condition dans laquelle il est inséré, si on implémente du else if

le || crée une condition alternative. si une des deux conditions est bonne, alors le code s'éxécute

on peut créer des conditions multiples, en créant des conditions entre différentes ||

utiliser le ! pour inverser la condition de validité

Casser une boucle

pour casser une boucle, on crée une boucle puis on y insère une condition qui quand elle est remplie arrête la boucle

on casse alors la boucle avec break

Continuer la boucle

le continue dans une boucle permet de casser la boucle et de recommencer

ici ça efface le 0 de la boucle

Switch

switch permet de créer plusieurs conditions, sans utiliser un else if

on crée les conditions avec case, on finit avec : , puis on écrit le code que l'on veut si la condition est remplie

le switch ne compare que des valeurs strictement identiques, pas possible de comparer un nombre avec une string

Scope

une variable peut être appelé dans une méthode ou en dehors, elle a alors un scope global, cela ne vaut que si elle a été créée en dehors d'un block

si une variable est créée dans un block ou modifie une variable dans le block, elle n'existe que dans le block et la valeur modifiée ne vaut que dans le block

en cas de block dans un block, la valeur de la variable est celle qui est la plus récente du block appelé

cela vaut pour les variables let et const
